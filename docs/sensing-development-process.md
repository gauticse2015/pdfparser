# Sensing / struggle development process

## Rule (non-negotiable)

```
fixture + gold  →  BASELINE capture  →  diagnose  →  generic fix  →  re-measure
     ↑                                                              │
     └──────── only then mark solved_regression ←──────────────────┘
```

**Never** implement a fix without:

1. An owned synthetic fixture (or existing hard/precision doc) that fails **gold**
2. A recorded baseline (shapes / cell F1 / detect F1) **before** the change
3. A written hypothesized root cause

**Never** copy ICDAR (or other competition) PDFs into the corpus.

## Suite roles

| Role | Meaning | Suite field |
|------|---------|-------------|
| `open_struggle` | Failing today; active development target | `development_role` in GT |
| `solved_regression` | Was a struggle; must stay green | same |
| precision/hard free wins | Guardrails, not struggle | not in open list |

## Commands

```bash
# regenerate sensing PDFs
python benchmark/scripts/generate_sensing_corpus.py   # 90–92 solved
# 93–95 currently generated by scripts; fold into generate_sensing_corpus when convenient

# baseline / scoreboard (all sensing docs)
python benchmark/scripts/run_accuracy_benchmark.py \
  --suite regression_sensing \
  --libs pdfparser,pdfplumber,camelot_lattice,camelot_auto \
  --tag sensing

# machine-readable open-struggle snapshot
cat benchmark/results/sensing_struggle_baseline.json
```

## Judging progress

- **Primary:** per-doc gap on `open_struggle` IDs (shape, cell F1, detect F1)
- **Secondary:** suite mean (includes solved docs — can look artificially high)
- **Guardrails:** `regression_hard` + `regression_precision` must not regress

## Historical note

Phases fixing 90/91/92 and 54 happened partly **without** freezing a formal pre-fix baseline file for 54. That was process debt. From this document forward, open fixtures are baselined first.
